## Short answer

The Regents public corpus gives humans and agents stable pages for the main Regents concepts, glossary terms, source cards, and build notes.

## Canonical definition

The corpus is the public reading layer for Regents. It explains the product family without replacing product contracts, source cards, or code documentation.

## Why it matters

Agents and answer engines need stable URLs, clear definitions, and status labels.

## How it works in Regents

Start with [What is Regents?](/learn/what-is-regents/), then read the source cards for each owned surface.

## What this does not claim

The corpus explains public facts. It does not define shipped API or CLI behavior.
